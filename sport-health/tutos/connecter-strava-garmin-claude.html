Connecter Strava & Garmin à Claude Code

Guide pas-à-pas pour connecter tes comptes sport via MCP

🏃 ⌚ 🤖

Prérequis

  • Claude Code installé — doc officielle
  • Node.js 18+ — vérifie avec node --version
  • Python 3.10+ et uv — pip install uv ou brew install uv
  • Un compte Strava et/ou Garmin Connect
🏃

Strava

Activités & Routes ›

1 Créer une application Strava API

Va sur strava.com/settings/api et crée une application :

  • Application Name : Claude Code MCP
  • Category : Training Analysis
  • Website : http://localhost
  • Authorization Callback Domain : localhost

Note ton Client ID et Client Secret.

L'app Strava API est gratuite. Elle te donne un accès en lecture à tes propres données.

2 Installer le serveur MCP

claude mcp add strava -- npx -y @r-huijts/strava-mcp-server

3 Authentification

  1. Lance claude dans ton terminal
  2. Demande quelque chose qui utilise Strava (ex : "mes dernières activités Strava")
  3. Le serveur ouvre ton navigateur pour autoriser l'accès
  4. Autorise l'app sur Strava
  5. Les tokens sont sauvegardés dans ~/.config/strava-mcp/config.json
Test : Dans Claude Code, demande "Donne-moi mes 5 dernières activités Strava"
⌚

Garmin Connect

Données, Santé & Entraînements ›

1 Pré-authentification

Il faut d'abord s'authentifier une fois manuellement :

uvx --python 3.12 --from git+https://github.com/Taxuspt/garmin_mcp garmin-mcp-auth

Le script demande :

  1. Email — ton email Garmin Connect
  2. Password — ton mot de passe
  3. Code MFA — le code à 6 chiffres si tu as la double authentification

Les tokens OAuth sont sauvegardés dans ~/.garminconnect/.

2 Installer le serveur MCP

claude mcp add garmin -- uvx --python 3.12 --from git+https://github.com/Taxuspt/garmin_mcp garmin-mcp

3 Ce que tu peux faire

Lecture de données :

  • Consulter tes activités (détails, splits, FC, météo...)
  • Lire tes données de santé (sommeil, stress, Body Battery, SpO2)
  • Voir ton profil, tes records personnels, ton score d'endurance
  • Gérer tes pesées et ta composition corporelle

Gestion des entraînements :

  • Créer des séances structurées (running, natation piscine, vélo...) avec upload_workout
  • Planifier des séances dans le calendrier Garmin avec schedule_workout
  • Supprimer des séances avec delete_workout
  • Consulter les entraînements planifiés avec get_scheduled_workouts
Un seul MCP suffit pour tout. Le paquet garmin-workouts-mcp n'est pas nécessaire — upload_workout offre un contrôle plus fin (repeat groups, repos manuels lap button, allures précises, taille de piscine, etc.).
Envoi vers la montre : les séances créées via l'API apparaissent dans ta bibliothèque Garmin Connect, mais ne sont pas envoyées automatiquement sur la montre. Pour les voir dans la liste d'entraînements de ta montre, clique sur "Envoyer vers l'appareil" depuis la page web ou l'app Garmin Connect.
Test : Dans Claude Code, demande "Donne-moi mon résumé d'activité de la semaine" ou "Crée un entraînement fractionné : 20' EF puis 6x(1' vite / 1' récup) puis 10' retour au calme"

Vérifier que tout est connecté

claude mcp list

Tu devrais voir :

garmin  uvx --python 3.12 --from [...]     ✓ Connected
strava  npx -y @r-huijts/strava-mcp-server ✓ Connected

Troubleshooting

›
ProblèmeSolution
Not authenticated (Garmin) Relance garmin-mcp-auth pour rafraichir les tokens
Invalid token (Strava) Supprime ~/.config/strava-mcp/config.json et relance Claude Code
Le serveur ne démarre pas Vérifie Node.js (≥ 18) et Python (≥ 3.10)
Disconnected dans mcp list claude mcp remove <nom> puis réinstalle

Exemples de prompts

"Analyse ma charge d'entraînement des 4 dernières semaines"
"Compare mes allures sur mes 3 dernières sorties longues"
"Crée un entraînement fractionné : 20' EF puis 6x(1' vite / 1' récup) puis 10' retour au calme"
"Planifie cette séance dans le calendrier pour lundi prochain"
"Quel est mon temps de sommeil moyen cette semaine ?"
"Génère un plan d'entraînement basé sur mes données récentes"
